Basic View

Views in Kudu are implemented as AMD modules that must return a Controller

Controllers are plain Javascript objects but must implement an onInit method that returns a Ractive instance. Ractive is a library that binds HTML templates to data objects. To get started with Ractive see here. A tutorial on Ractive is available here.

Controllers have an associated HTML template, which are mustache based. Mustache is a logicless templating engine, and Ractive has great support for binding mustaches to data variables.

Let's look at our first Kudu Controller below, basic.js: 

define(function (require) { // A normal AMD define declaration

	var template = require("rvc!./basic"); // Import the Controller's HTML template, basic.html

	// Our controller object
	var basicCtrl = {};

	// We add the required onInit method that returns a Ractive view instance
	basicCtrl.onInit = function (options) {

		return new template(); // We create and return a new Ractive view from the template
	};

	return basicCtrl;
});

In line 3, we use the rvc plugin to load and compile the view.html template. The result of this plugin is a fully compiled Ractive object that can be instantiated using new.

In line 9 we implement the required onInit method that must return a Ractive view instance.

In line 11, we create a and return a new Ractive view instance from the compiled template from line 3.
